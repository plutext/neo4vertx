Bridging the gap between Vert.x and Neo4j

This example demonstrates how to use neo4vertx in conjunction with java in its default mode.

The storeNode() method is used to store data while the countData() method demonstrates how to read data from the neo4j database.

In this example cypher queries are used to store and retrieve data results from the embedded neo4j database. The event bus address "graph.cypher.query" is used to communicate with the neo4vertx module.

private void storeNode() {
     JsonObject obj = new JsonObject();
     String query = "CREATE (testentry:ingredient"
             + System.currentTimeMillis() + " {time : '"
             + System.currentTimeMillis() + "'})" ;
     obj.putString( "query" , query);
     logger.info( "Sending query: " + query);
     vertx.eventBus().send( "graph.cypher.query" , obj);
}

private void countData() {
     ResponseHandler responseHandler = new ResponseHandler();
     JsonObject obj = new JsonObject();
     String query = "MATCH (n) RETURN count(*);" ;
     obj.putString( "query" , query);
     logger.info( "Sending query: " + query);
     vertx.eventBus().send( "graph.cypher.query" , obj, responseHandler);
}

A custom response handler is used in order to retrive the response json from the event bus. The json is than unmarshaled using GSON. I used GSON in this example because it simplifies the way how to interact with the response object.

class ResponseHandler implements Handler<Message> {
     private static Logger logger = LoggerFactory
             .getLogger(ResponseHandler. class );
 
     @Override
     public void handle(Message event) {
         QueryResponse response = new Gson().fromJson(event.body().toString(),
                 QueryResponse. class );
         System.out.println( "Elements:"  + response.getData().get( 0 ).get( 0 ));
         logger.info( "Found " + response.getData().get( 0 ).get( 0 ) + " Elements" );
     }
}
